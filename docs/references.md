# References

RDLLM design references are maintained in two places:

- `docs/recent_research.md` for implementation-facing notes about current
  attribution, citation, provenance, provider API, policy, and standards work
- `paper/references.bib` for BibTeX references used by the manuscript
- `paper/rdllm_white_paper.md` for the public state-of-the-art synthesis that
  connects recent papers, provenance standards, policy sources, runtime
  examples, allocation, and product-readiness requirements

The release process requires adding primary sources when a new verifier,
provider mapping, policy rule, standard export, or attribution metric depends on
external work.

Key reference categories:

- source attribution and citation verification
- claim/evidence decomposition
- retrieval-augmented generation evaluation
- provenance, transparency, and verifiable credentials
- provider API response, citation, usage, safety, and streaming fields
- copyright, licensing, training-data transparency, and creator remuneration
- payment, settlement, and auditability standards
- production operation and public-sector readiness controls

Production-readiness references:

- NIST AI 600-1, Artificial Intelligence Risk Management Framework:
  Generative Artificial Intelligence Profile.
- NIST SP 800-218, Secure Software Development Framework (SSDF) Version 1.1.
- OWASP Top 10 for Large Language Model Applications 2025.
- SLSA, Supply-chain Levels for Software Artifacts.
