# Code of Conduct

Contributors are expected to keep discussion technical, evidence-based, and
respectful. Disagreement is acceptable; harassment, threats, personal attacks,
and bad-faith disruption are not.

Project decisions should be argued through requirements, tests, verifiers,
security impact, interoperability impact, and documented tradeoffs.

Maintainers may remove abusive comments, close disruptive issues, or restrict
participation when behavior materially harms the project.
